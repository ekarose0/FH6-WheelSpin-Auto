# FH6Auto

**Python + 이미지 인식 + 입력 자동화** 기반의 FH6 비주얼 스크립트 도구.  
**반복 맵 주행 / 일괄 차량 구매 / 슈퍼 뽑기 / 차량 제거 / 다중 모듈 연결 / 무한 반복 매크로**를 지원합니다.

> 본 도구는 Python 자동화 기술 교류 및 학습 목적으로만 사용해 주세요. 상업적 용도나 게임 밸런스를 훼손하는 용도로 사용하지 마세요.
> 본 도구 사용으로 인해 발생한 모든 결과(계정 정지, 오류, 손실 등 포함)는 사용자 본인이 부담해야 합니다.

## 프로젝트 개요

FH6Auto는 게임 화면의 자동 인식 및 프로세스 제어를 중심으로 설계된 데스크톱 자동화 도구입니다.  
본 프로젝트는 **이미지 인식**을 프로세스 가이드의 핵심으로 사용하여, 단순 키 입력 스크립트의 "맹목적 조작"으로 인한 제어 불가 상태를 최소화합니다.

도구는 다음 방식으로 자동 실행을 구현합니다:

- 스크린샷으로 현재 게임 상태 인식
- 목표 페이지 진입 여부 동적 판단
- 키보드 / 마우스 동작 트리거
- 오류 발생 시 실행 환경 자동 복구 시도

기존 단순 키 입력 스크립트에 비해 안정성, 적응성, 제어성이 더 뛰어납니다.

## 기능 모듈

### 1. 반복 맵 주행
- 메뉴 자동 진입
- 크리에이티브 허브 / EventLab 자동 전환
- 블루프린트 공유 코드 자동 입력
- 목표 차량 자동 매칭
- 레이스 자동 반복 시작
- 설정된 횟수만큼 반복 실행 지원

### 2. 일괄 차량 구매
- 차량 컬렉션 자동 진입
- 목표 브랜드 자동 탐색
- 지정 차량 자동 선택
- 자동 반복 구매
- 설정된 횟수만큼 일괄 실행 지원

### 3. 슈퍼 뽑기
- 차량 구매 프로세스 자동 진입
- 목표 차량 자동 탐색
- 업그레이드 / 숙련도 화면 자동 진입
- 스킬 매트릭스에 따라 스킬 자동 할당
- 스킬 포인트 소진 후 모듈 자동 종료 지원


### 4. 대형 반복 매크로
모듈 연결로 전체 파이프라인 구성 가능:

**맵 주행 → 차량 구매 → 뽑기 → 카운트 리셋 → 다음 라운드**

설정 가능 항목:
- 다음 모듈로 계속 진행 여부
- 3개 모듈 완료 후 반복 여부
- 총 반복 횟수
- 사실상 무한 매크로 실행


## 사용 방법

### 1. 시작 전 준비

#### 차량 준비
맵 주행에 사용할 **스바루 Impreza 22B-STi Version**을 먼저 구매하고 다음 준비를 완료하세요:

- **S2 900**으로 튜닝
- 해당 차량을 컬렉션에 추가
- 어떠한 랩(도색)도 적용하지 않기

#### 사용 전 확인 사항
- **필터, HDR 등 색상에 영향을 주는 설정 모두 끄기**
- 게임이 정상적으로 실행 중인지 확인
- 게임 언어를 **중국어 간체**로 설정
- 키보드 입력기를 **영문 키보드**로 전환
- 게임 내 설정: **자동 조향**, **오토 기어**, 난이도 **언터처블** 권장
- 게임 화면을 최대한 안정적으로 유지
- 이미지 인식 결과에 영향을 줄 수 있으므로 다른 창으로 임의 전환하지 않기

---

### 2. 매개변수 설정

프로그램 메인 화면에서 설정 가능:

- 맵 주행 횟수
- 차량 구매 횟수
- 뽑기 횟수
- 블루프린트 공유 코드
- 총 반복 횟수
- 다음 모듈 연결 여부
- 3모듈 대형 반복 활성화 여부
- 자동 재시작 메커니즘 활성화 여부
- 자동 재시작 명령어

---

### 3. 스킬 경로 설정

"슈퍼 뽑기" 모듈 영역에서:

- 방향 버튼 클릭으로 스킬 경로 추가
- "매트릭스 초기화" 클릭으로 경로 리셋
- 파란색 칸은 현재 스킬 이동 경로를 나타냄

---

### 4. 단일 모듈 실행

모든 모듈을 독립적으로 실행 가능:

- 반복 맵 주행
- 일괄 차량 구매
- 슈퍼 뽑기

프로그램은 해당 모듈부터 실행을 시작합니다.

---

### 5. 연결 프로세스 실행

화살표 위치의 "계속" 옵션을 체크하면 모듈이 자동으로 연결됩니다. 예:

- 맵 주행 완료 후 차량 구매로 계속
- 차량 구매 완료 후 뽑기로 계속

`LOOP -> 반복 초기화`를 추가로 체크하면, 3개 모듈 완료 후 자동으로 다음 라운드에 진입합니다.

---

### 6. 스크립트 중지

다음 방법으로 중지 가능:

- 화면의 중지 버튼 클릭
- 키보드 **F8** 누르기

중지 후 프로그램은 다음을 최대한 수행합니다:
- 현재 스레드 중지
- 눌려진 모든 키 해제
- 메인 화면 상태 복원

---

### 7. 안전 일시정지 및 재개

다음 방법으로 안전 일시정지를 요청할 수 있습니다.

- 화면의 일시정지 버튼 클릭
- 키보드 **F9** 누르기

프로그램은 현재 작업을 즉시 중단하지 않고:

- 현재 진행 중인 작업 완료
- 자동으로 메뉴 복귀
- 현재 진행 상태 유지
- 일시정지 상태 진입

을 수행합니다.

예를 들어:

- 현재 레이스 진행 중
- 현재 차량 구매 중
- 현재 스킬 경로 실행 중

인 경우에도 현재 1회 작업이 완료된 뒤 일시정지됩니다.

#### 일시정지 요청 취소

실수로 일시정지를 요청한 경우:

- **F9** 다시 누르기
- 일시정지 버튼 다시 클릭

으로 요청을 취소할 수 있습니다.

#### 작업 재개

일시정지 상태에서는:

- **F9** 누르기
- 재개 버튼 클릭

으로 작업을 다시 시작할 수 있습니다.

재개 시 현재 모듈부터 다시 진입하며, 이미 완료된 작업 수는 그대로 유지됩니다.

## 이미지 템플릿 설명

프로젝트는 주요 인식 템플릿 교체를 지원합니다. 일반적인 커스텀 이미지는 다음과 같습니다:

- `CCbrand.png`: 컨슈머블 차량 브랜드
- `consumablecar.png`: 스킬 할당 및 슈퍼 뽑기에 사용되는 컨슈머블 차량
- `newCC.png`: 신차 표시용 컨슈머블 차량
- `skillcar.png`: 맵 주행 시 스킬 포인트를 획득하는 데 사용되는 차량
  > 해당 차량은 반드시 컬렉션에 추가되어 있어야 하며, 이미지에서 컬렉션 아이콘이 표시되어야 합니다.

### 이미지 리소스 특징
프로젝트의 `images` 템플릿 이미지는 다음을 지원합니다:

- 프로그램 디렉토리 외부 `images`를 우선 읽기
- 외부에 없을 경우 내장 리소스 읽기
- 시작 시 템플릿 이미지를 외부 디렉토리에 자동 추출

이 방식의 장점:

- 사용자가 직접 템플릿을 교체 가능
- 다양한 화질 / 해상도 / UI 상태에 맞게 인식 미세 조정 가능
- 유지보수 및 업데이트가 더 편리

## 오류 방지 및 복구 설명

실행 중 다음 상황이 발생한 경우:

- 이미지 인식 실패
- 현재 화면 비정상
- 프로세스 중단
- 게임 강제 종료

프로그램은 다음을 우선 시도합니다:

1. 게임 프로세스 확인
2. 게임 창 활성화
3. 프리라이드 / 메뉴로 복귀
4. 계속 실행 가능한 상태로 자동 복구

자동 재시작이 활성화된 경우:

- 미리 설정된 명령어로 게임 재시작 시도
- 계속 화면을 인식한 후 프로세스 복원

> 자동 복구가 100% 성공을 보장하지는 않지만, 보호 기능이 없는 스크립트보다 안정성이 높습니다.

## 프로젝트 특징

### Python 이미지 인식 기반
본 프로젝트는 단순한 고정 좌표 클릭 도구가 아니며, 다음 기술을 결합한 화면 인식을 수행합니다:

- `pyautogui` 스크린샷
- `opencv-python (cv2)` 템플릿 매칭
- `numpy` 이미지 처리

현재 게임 화면을 인식한 후 다음 동작을 결정하므로, 단순 스크립트보다 높은 적응성과 제어성을 갖습니다.

### "맹목적 조작" 위험 최소화
전통적 스크립트의 일반적인 문제:

- 페이지 상태를 판단하지 않음
- 기계적으로 계속 키 입력
- 화면이 멈추면 완전히 꼬임

본 프로젝트는 이미지 인식으로 다음을 확인합니다:

- 현재 프리라이드 화면인지 여부
- 메인 메뉴 진입 여부
- 지정 버튼 발견 여부
- 업그레이드 페이지 진입 여부
- 계속하기 / 환영 화면 / 재시작 화면 표시 여부

올바른 상태가 인식된 경우에만 후속 동작을 실행합니다.

### 다층 오류 방지 메커니즘
프로젝트 내에 다중 보호 로직이 포함되어 있습니다. 예:

#### 1. 상태 검증
핵심 단계 전에 확인:
- 메뉴 진입 성공 여부
- 목표 페이지 인식 여부
- 지정 이미지 템플릿 발견 여부
- 여전히 실행 상태인지 여부

#### 2. 중단 복구
모듈 실행 실패 시 시도:
- 게임 프로세스 존재 여부 확인
- 게임 창 자동 활성화
- 메뉴 / 프리라이드로 자동 복귀 시도
- 중단된 모듈부터 나머지 프로세스 계속 실행

### 3. 자동 재시작 메커니즘

게임 프로세스가 종료되었거나 복구가 불가능한 상태가 감지되면:
- 사용자 지정 시작 명령 실행
- 게임 자동 실행
- 시작 화면 자동 인식
- Continue 화면 자동 인식
- 메뉴 진입 확인
- 중단된 작업 자동 복구 시도

를 수행합니다.

가능한 경우 작업을 처음부터 다시 시작하지 않고, 중단된 모듈부터 계속 실행합니다.

#### 4. 키 강제 해제
스크립트 중지 시 능동적으로 해제:
- 방향키
- Enter / Esc / Backspace / Space
- 계속 눌려진 W 키 등

비정상 종료로 인한 "키 잠김" 현상을 방지합니다.

### 5. VRAM 부족 자동 복구 [한국어 이미지가 없어 미지원]

장시간 실행 중 VRAM 부족 관련 오류 화면이 감지되면:

- 오류 화면 자동 인식
- 게임 프로세스 자동 종료
- VRAM 해제를 위한 대기
- 게임 자동 재실행
- 실행 중이던 작업 자동 복구 시도

를 수행합니다.

장시간 무인 실행 시 발생할 수 있는 VRAM 부족 상황에 대응하기 위한 보호 기능입니다.

### 무한 반복 매크로 지원
편집 가능한 소형 매크로 파이프라인으로 이해할 수 있습니다:

- 먼저 맵 주행으로 수익 획득
- 그다음 자동 차량 구매
- 이어서 자동 숙련도 스킬 할당으로 뽑기 획득
- 완료 후 다음 라운드 진입

총 반복 횟수 설정과 함께 장시간 무인 실행이 가능합니다.
반복 횟수를 충분히 높게 설정하면 "무한 반복 매크로"로 간주할 수 있습니다.

### 스킬 매트릭스 커스터마이징
슈퍼 뽑기 모듈은 스킬 포인트 경로를 수동으로 설정하는 것을 지원합니다.
사용자는 UI의 방향 버튼을 통해 다음을 커스터마이즈할 수 있습니다:

- 위
- 아래
- 왼쪽
- 오른쪽

스킬 경로 매트릭스를 형성하여 다양한 차량의 숙련도 트리에 맞게 조정할 수 있습니다.

## 기술 구현

본 프로젝트는 주로 다음 Python 기술 스택을 기반으로 합니다:

- **customtkinter**: 모던 데스크톱 UI
- **opencv-python**: 템플릿 매칭 / 이미지 인식
- **numpy**: 이미지 배열 처리
- **pyautogui**: 스크린샷 및 기본 자동화
- **pydirectinput**: 게임 환경에 더 적합한 입력 시뮬레이션
- **ctypes / win32gui**: 창 활성화, DPI 적응, 저수준 입력
- **threading**: 백그라운드 작업 실행
- **PIL**: 리소스 이미지 로드

## 적용 환경 특징

- Windows 환경 대상
- 다양한 해상도에서 템플릿 스케일링 매칭 지원
- DPI 인식 처리로 고배율 디스플레이의 좌표 오차 감소
- 메인 화면 최소화 실행 로직 지원
- 단축키 **F8** 긴급 중지 지원
- 단축키 **F9** 안전 일시정지 / 재개 지원
- 자동 복구 메커니즘 지원
- 자동 재시작 메커니즘 지원
- VRAM 부족 자동 복구 지원
